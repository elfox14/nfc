# قائمة إطلاق المحرر إلى الإنتاج

هذه القائمة مطلوبة بعد دمج أي تحديث يؤثر في المحرر أو الحفظ أو النشر.

## 1. النشر

- تأكد أن Render نشر أحدث commit من `main`.
- افتح `/api/health` وتحقق من `status`, `version`, `release`.
- افتح `/readyz` وتأكد أن الحالة `ok` وأن فحصي قاعدة البيانات والتخزين `ready`.
- مزامن ملفات الواجهة إلى `public_html/nfc/` عند استخدام cPanel/LiteSpeed.
- نفذ تحديثًا قسريًا وتأكد أن Service Worker يستخدم cache `v6`.

## 2. رحلة المستخدم الأساسية

نفذها بالعربية والإنجليزية:

1. تسجيل الدخول بالبريد.
2. تسجيل الدخول عبر Google.
3. إنشاء تصميم جديد.
4. تعديل الاسم والوظيفة ووسائل الاتصال.
5. رفع شعار وصورة.
6. تعديل ترتيب الطبقات والموضع والحجم والدوران والشفافية.
7. الحفظ وإغلاق الصفحة.
8. إعادة فتح التصميم والتأكد من بقاء جميع التعديلات.
9. المعاينة ثم النشر.
10. فتح رابط Viewer ومقارنته بالمحرر.
11. اختبار PNG وPDF وVCF وQR.

## 3. اختبارات فقدان البيانات

- عدّل حقلًا وتأكد من ظهور «تغييرات غير محفوظة».
- حاول إغلاق التبويب وتأكد من ظهور تحذير المتصفح.
- اقطع الإنترنت، عدّل التصميم، وتأكد من «محفوظ محليًا».
- أعد تحميل الصفحة وتأكد من ظهور خيار استعادة النسخة المحلية.
- أعد الاتصال وتأكد من إعادة محاولة الحفظ السحابي.
- نفذ تعديلات متتابعة سريعة وتأكد أن آخر تعديل هو المحفوظ.
- اختبر فشل API وتأكد أن النسخة المنشورة السابقة لا تتأثر.

## 4. المتصفحات والأجهزة

- Chrome Desktop.
- Chrome Android.
- Safari Desktop/WebKit.
- Safari iPhone.
- شاشة صغيرة ومتوسطة وكبيرة.
- الوضع الفاتح والداكن.

## 5. شروط الموافقة

لا يُعتمد الإصدار إلا إذا:

- نجحت اختبارات Jest وESLint وnpm audit.
- نجحت Playwright في Chromium وWebKit للهاتف وسطح المكتب.
- تطابق Viewer مع المحرر.
- لم تظهر أخطاء JavaScript حرجة.
- نجحت الاستعادة المحلية والحفظ بعد عودة الشبكة.
- عاد `/readyz` بحالة 200.

## التراجع

عند ظهور خلل حرج:

1. أوقف نشر الإصدار الجديد أو أعد نشر آخر commit مستقر.
2. لا تحذف التصاميم أو النسخ المحلية للمستخدمين.
3. احتفظ بتقرير Playwright وسجلات `client-error` مع رقم `release`.
4. أصلح المشكلة في Pull Request مستقل وأعد تنفيذ القائمة كاملة.
