# إطلاق مساحات عمل الفريق ودورة مراجعة التصميم

الإصدار المستهدف: `2026.07.19-phase11.0`

إصدار Service Worker: `v14`

## ملفات الخادم على Render

يجب أن يتضمن النشر أحدث نسخ الملفات التالية:

- `server.js`
- `auth-middleware.js`
- `utils/database.js`
- `utils/workspace-access.js`
- `routes/workspaces.routes.js`
- `routes/workspace-design-bridge.routes.js`
- `routes/workspace-publication-guard.routes.js`
- `routes/design-versions.routes.js`
- `routes/seo.routes.js`

ينشئ الخادم تلقائيًا مجموعتي MongoDB:

- `workspaces`
- `designReviews`

ويضيف فهارس لمساحة العمل وحالة دورة المراجعة داخل مجموعة `designs`. لا توجد ترحيلات يدوية مطلوبة.

بعد اكتمال نشر Render ومزامنة ملفات الواجهة شغّل:

npm run verify:production

ينفذ الفاحص 26 فحصًا تشمل:

- لوحة التحكم العربية والإنجليزية.

- `/healthz` و`/readyz` و`/api/health`.

## رحلة الاعتماد الأساسية

1. أنشئ مساحة عمل من Dashboard.
2. أضف مستخدمًا مسجلًا بدور `editor`.
3. أضف مستخدمًا مسجلًا آخر بدور `reviewer`.
4. اربط تصميمًا محفوظًا بمساحة العمل.
5. افتح التصميم بحساب المحرر وعدّل الاسم أو اللون.
6. تأكد أن حالة التصميم أصبحت `draft` حتى لو كان معتمدًا أو منشورًا قبل التعديل.
7. أضف تعليقًا عامًا وتعليقًا مرتبطًا بعنصر محدد.
8. أرسل التصميم للمراجعة.
9. افتحه بحساب المراجع واطلب تعديلات مع ملاحظة واضحة.
10. تأكد أن المحرر يستطيع التعديل وأن الحالة تعود إلى `draft`.
11. أعد الإرسال للمراجعة ثم اعتمد التصميم.
12. تأكد أن النشر غير متاح قبل الاعتماد.
13. انشر التصميم بحساب المالك أو المدير.
14. افتح Viewer دون تسجيل دخول وتأكد أن التصميم المنشور فقط هو المتاح.

## اختبارات الخصوصية والأمان

- الزائر غير المسجل لا يستطيع فتح تصميم حالته `draft` أو `in_review` أو `changes_requested` أو `approved`.
- عضو مساحة العمل يستطيع معاينة التصميم قبل النشر بعد تسجيل الدخول.
- المستخدم خارج مساحة العمل لا يستطيع قراءة التصميم أو التعليقات أو الإصدارات.
- `viewer.html` و`/api/get-design/:id` يعيدان عدم العثور للمستخدم غير المصرح له.
- التصميم غير المنشور لا يظهر داخل `sitemap.xml` حتى إذا كان `sharedToGallery=true`.
- دور `viewer` لا يستطيع الحفظ أو اعتماد التصميم أو النشر.
- دور `reviewer` يستطيع التعليق والاعتماد وطلب التعديلات، ولا يستطيع تعديل التصميم.
- دور `editor` يستطيع التعديل والإرسال للمراجعة، ولا يستطيع اعتماد التصميم أو نشره.
- المالك والمدير فقط يستطيعان إدارة الأعضاء والنشر بعد الاعتماد.
- أي حفظ أو استعادة إصدار لتصميم داخل مساحة العمل يلغي الاعتماد السابق ويعيد الحالة إلى `draft`.

## التراجع

عند ظهور خلل حرج:

1. أعد نشر آخر commit مستقر على Render.
2. أعد ملفات الواجهة السابقة إلى `public_html/nfc/`.
3. ارفع رقم كاش Service Worker عند إصدار الإصلاح.
4. لا تحذف مجموعات `workspaces` أو `designReviews` أو `designVersions`.
5. احتفظ بتقرير Playwright و`production-verification.json` وسجلات `/api/client-error`.
